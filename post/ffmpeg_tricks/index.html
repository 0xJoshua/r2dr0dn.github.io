Ffmpeg Tricks

Upscale Video

Upscale Video To 4k

ffmpeg -i input.mp4 -vf scale=3840x2160:flags=lanczos output.mp4

Reduce File Size

Time in Seconds / File Size in GigaBits –> Used as bitrate

ffmpeg -i input.mp4 -b $(S/G) output.mp4

Mp4 To VP9 (Webm)

Constate Quality

H264 Quality Options from 0 to 64 (crf 28 best)

ffmpeg -i in.mp4 -c:v libvpx-vp9 -crf 28 -b:v 0 out.webm

Contrained Quality With Bitrate

A popular option from h264 encoding, a quality value is given between 0 and 63 with lower values meaning higher quality. It is recommended to use a value between 15 and 35. To use constant quality -b:v must be set and at zero: -b:v 0

ffmpeg -i in.mp4 -c:v libvpx-vp9 -crf 28 -b:v 1500k out.webm
  • With max and mix bitrate
ffmpeg -i in.mp4 -c:v libvpx-vp9 -minrate 400k -b:v 1500k -maxrate 2200k out.webm

Lossless Encoding 

ffmpeg -i in.mp4 -c:v libvpx-vp9 -lossless 1 out.webm

Speed And Quality

  • -deadline is the main option for VP9 encoding speed, as per media encoding lore the slower options do a more efficient encode.
  • good (default) is the recommended.
  • best has the finest compression (slow).
  • realtime live and fast encoding.
  • -cpu-used is how efficient the compression will be, when using a deadline of good or best the value that can be set must be between 0 and 5.

Default it is 0, 1 or 2 will increase speed but cost in quality and rate control. A value of 4 or 5 disables rate-distortion optimization thus having a greater impact on quality. To further utilize modern CPU’s when encoding row-based multithreading can be turned on with -row-mt 1 this can help dramatically. More encoding speed can be achieved with specifying the CPU threads to use when encoding with -threads an example being: -threads 16.

ffmpeg -i in.mp4 -c:v libvpx-vp9 -row-mt 1 -threads 16 -cpu-used 2 -deadline good -crf 21 -b:v 0 out.webm

Personal Preference 

ffmpeg -i input.mp4 -c:v libvpx-vp9 -row-mt 1 -threads 48 -cpu-used 5 -deadline good -crf 28 -b:v 0 out.webm

Adding Audio To A Video 

ffmpeg -i video.mp4 -i audio.mp3 -c copy -map 0:v:0 -map 1:a:0 output.mp4

Cut/Split a video in N segments

Change N to the number of minutes / hours / seconds

ffmpeg -i videotocut.mp4 -c copy -map 0 -segment_time 00:0N:00 -f segment segment%03d.mp4

Change Frame rate

  • Change frame rate with ffmpeg with -filter:v fps=fps=25
# 60 Fps Video Make it to 30 fps video
ffmpeg -i inputvideo.mp4 -filter:v fps=fps=30 inputvideo_30fps.mp4

Change fps while encoding 

ffmpeg -i inputvideo.mp4 -filter:v fps=fps=30 -c:v libx264 -preset fast -crf 23 output.mp4

Add text watermark to video

adding a text watermark with drawtext from ffmpeg in bottom left

ffmpeg -i "input.mp4" -vf "drawtext=text='a watermark':x=10:y=H-th-10:fontfile=/pathto/font.ttf:fontsize=10:fontcolor=white:shadowcolor=black:shadowx=2:shadowy=2" "output.mp4"

Video Compression

Using ffmpeg --preset and -crf to determine best output size, preset is the encoding the compression rate, the slower the better therefor more quality in a smaller size. veryslow slower slow CRF means constant rate factor, essentially it is the bitrate amount without it being specific. Lower value = higher quality. 0 is lossless but a value of 17 or 18 is still visually lossless. 21 or 22 i find to be best but once you start going above 25 the quality drops away.

ffmpeg -i inputfilename.mp4 -c:v libx264 -preset veryslow -crf 21 outputfilename.mp4
  • Something little quicker
ffmpeg -i inputfilename.mp4 -c:v libx264 -preset slower -crf 22 outputfilename.mp4

Speed/Slow Down Video

To Speed Up .

ffmpeg -i input.mkv -filter:v "setpts=0.5*PTS" output.mkv

Using blending

ffmpeg -i typingspeed.mkv -vf "tblend=average,framestep=2,tblend=average,framestep=2,setpts=0.25*PTS" -r 60 output.mkv

Minterpolate Using ffmpeg ==> creates a 96 fps video

ffmpeg -i input -filter:v minterpolate -r 96 output

then speed it up

ffmpeg -i input -vf "tblend=average,framestep=2,tblend=average,framestep=2,tblend=average,framestep=2,tblend=average,framestep=2,setpts=0.25*PTS" -r 96 -b:v 30M -crf 10 -an output

move commands

fmpeg -i input.avi -filter:v "minterpolate='fps=120',setpts=4*PTS" output.avi
ffmpeg -i typingspeed.mkv -filter:v "minterpolate='mi_mode=mci:mc_mode=aobmc:vsbmc=1:fps=120'" output.mkv

Cut Video 

ffmpeg -i comeback-2022-02-24_17.38.33.mkv -ss 00:00:08 -to 00:03:47 -async 1 -c copy comeback.mp4
  • to cut microseconds add (dot.XXX) 00:00:08.100

30 fps to 60fps 

Four step search
ffmpeg -i input.mp4 -filter:v "minterpolate='mi_mode=mci:mc_mode=aobmc:vsbmc=1:fps=60:me=fss'" out.mp4

Diamond search
ffmpeg -i input.mp4 -filter:v  "minterpolate='mi_mode=mci:mc_mode=aobmc:vsbmc=1:fps=60:me=ds'" out.mp4
